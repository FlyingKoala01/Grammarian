# Roadmap

## Phase 1 - Foundation
Goal: establish a clean technical base and one end-to-end learning loop.

Deliverables:
- repo setup with pnpm and TypeScript
- frontend shell with Tailwind and shadcn/ui
- Express backend with routing and validation
- ORM and initial database schema
- authentication
- first study session flow
- simple progress persistence

## Phase 2 - Core Learning Experience
Goal: make studying and revision genuinely useful.

Deliverables:
- multiple exercise types
- revision queue
- answer submission workflow
- correction screens
- progress dashboard
- deterministic checking for simple exercises

## Phase 3 - LLM Enhancement
Goal: use the LLM where it clearly improves learning value.

Deliverables:
- diversified exercise generation
- open-answer correction
- explanation generation
- ask-the-tutor interface
- prompt/version management
- resilience around provider errors

## Phase 4 - Quality and Mobile Readiness
Goal: improve reliability and prepare Android export.

Deliverables:
- stronger test coverage
- analytics on study flow completion
- performance improvements
- responsive polishing
- Capacitor integration
- Android validation

## Prioritization Rules
Always prioritize:
1. learner value
2. clarity of implementation
3. maintainability
4. reliability

Avoid spending early time on:
- broad theming systems
- speculative abstractions
- non-essential gamification
- low-value technical novelty
